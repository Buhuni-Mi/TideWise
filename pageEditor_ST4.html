User Profile

Technical Description

The profile page follows the brief closely by using JavaScript prompts instead of an HTML form. I broke the process into three steps and used the page itself as the output area. As the user completes prompts, the profile sections update without reloading the page.

I stored the profile data in JavaScript objects and used functions to update the progress bar and summary area. I also allowed skipped steps to be returned to later, which was one of the main requirements.

Accessibility

The instructions are kept visible on the page, the progress text updates clearly, and the output areas are separated into labelled boxes so users can understand what has already been completed.

Sitemap

Technical Description

The sitemap is built with SVG rather than a normal image. Each node links to the correct page and the diagram shows the site hierarchy from the splash page down to the content pages. I kept the design simple so it stays readable on different screen sizes.

I used SVG shapes, text labels, and links, with hover and focus styling to make the diagram interactive. This matched the specification better than using a screenshot or exported diagram image.

Accessibility

The SVG uses a readable label, focusable links, and visible hover and focus states. I also kept the text labels clear enough that the diagram still works as navigation rather than decoration only.

Content Page

Technical Description

My content page is about student pledges and sustainable ocean-friendly lifestyles. I focused on everyday habits because they fit the project theme and also connect well with the profile page.

The page uses internal navigation, a visible top link, and supporting images that reinforce the topic without taking over the page.

Accessibility

I used clear section headings, internal navigation, and meaningful image alt text. The content is split into short sections so the long-form page is easier to read and navigate.

Challenges and Lessons Learned

The profile page took the most time because it had to use prompts, progress tracking, skipped steps, and page updates together. The sitemap was also a bit tricky because it had to be true SVG and still stay readable. I kept both pages simple so the technical features were clear rather than flashy.

Accessibility Considerations

Accessibility on my pages was more about interaction flow. On the profile page, the instructions had to be clear because the prompts happen step by step and the output changes progressively. On the sitemap page, keyboard access and visible focus matter more because users need to move between SVG links. On the content page, I focused on readable section order and meaningful image alt text.

Compliance

For my work, JANET compliance meant publishing content responsibly and making sure the interactive parts stayed appropriate for a coursework site. I kept the profile questions relevant to the theme, avoided collecting unnecessary personal information, and made the sitemap purely navigational rather than decorative. I also kept all content original and suitable for an academic submission.
