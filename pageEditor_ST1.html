Home Page

Technical Description

I used the shared template so the header, navigation, and footer stayed consistent with the rest of the site. Inside the main section, I arranged the mission content into clear cards and added links to all four content pages so users can understand the site quickly.

The structure mainly uses `header`, `nav`, `main`, `section`, `article`, and `footer`. For styling, I reused the global card styles and added a small embedded style block for home-page-only emphasis so the shared stylesheet stayed clean.

Accessibility

I used clear heading order, strong colour contrast, a skip link, and visible focus styles. The hero and mission sections were kept short and readable, which helps keyboard and screen-reader users move through the page more easily.

Gallery Page

Technical Description

For the gallery, I kept the layout simple: four thumbnails, a title for each item, and a modal-style extended view. JavaScript opens the correct image, inserts the matching text, closes the panel, and applies the temporary theme and font changes inside the expanded view.

I used a thumbnail grid in HTML and linked it to DOM event listeners in JavaScript. Hover and focus feedback make it obvious that the images are interactive.

Accessibility

The images use descriptive alt text, the close button is easy to find, and keyboard focus remains visible when the modal controls are used. I also kept the text in the expanded view readable instead of relying on the image alone.

Content Page

Technical Description

My content page focuses on beach cleanups. I split the page into several sections with internal anchor links because the page is longer than one screen, and I added a visible go-to-top link and supporting images.

The content is organised with one `h1` and multiple `h2` sections. The layout uses a side contents area and a main reading area so the page is easier to scan.

Accessibility

I used descriptive image alt text, clear section headings, and internal links that help users jump directly to the relevant topic. The reading layout also reduces long uninterrupted text blocks.

Challenges and Lessons Learned

The hardest part for me was making the shared layout reusable for everyone else without making it too rigid. The gallery also needed JavaScript interaction, but I did not want it to feel over-designed. I solved this by keeping the page structure simple and then adding only the required interactive parts.

Accessibility Considerations

Because I handled the template and two visual pages, I focused most on structure and navigation clarity. I used alt text on images, clear headings, strong colour contrast in the navigation, and visible hover and focus states so users can tell what is clickable. On the gallery page, the close button and image controls were kept obvious so users are not trapped inside the expanded view.

Compliance

For my pages, JANET compliance mainly meant using appropriate educational content, avoiding misleading claims, and making sure all media and ideas used in the site stayed relevant to the project. I kept the homepage wording original instead of copying content from the UN website and used locally stored files rather than embedding random third-party content.
