Splash Screen

Technical Description

I designed the splash page as a short introduction rather than a complex landing page. It includes the title, group names, background image, a loader, a countdown, and a skip button. I kept it visually similar to the rest of the site even though this page does not use the shared template.

The automatic redirect is handled with a meta refresh tag, while the countdown text is updated with JavaScript. I used a simple full-screen layout and overlay styling to keep the text readable over the image.

Accessibility

I made sure the text stays readable on top of the background image by using contrast and spacing. The page also includes a direct skip link to `home.html` so users are not forced to wait for the redirect.

Action Impact Simulator

Technical Description

The AIS page includes six action cards, each with points. JavaScript tracks the selected cards, adds the score, updates the summary, and changes the background image depending on the impact level. I kept the rules simple so the interaction is easy to explain in the demo.

I used HTML cards for the actions, embedded CSS for page-specific layout, and JavaScript for score calculation and feedback. The selected state is clear so users can see which actions are currently counted.

Accessibility

The action cards can receive keyboard focus, the score updates are visible in text, and the interactive states are clear enough that users can tell when a choice has been applied. I also kept the headings and score label easy to scan.

Content Page

Technical Description

My content page is about plastic reduction habits for students. I focused on realistic habits that people can actually follow, so the writing stays practical instead of sounding too general.

I used section headings, internal links, a contents area, and supporting images. The page follows the same structure as the other content pages so the site feels unified.

Accessibility

The page uses a clear heading structure, descriptive alt text, and internal links that help readers move directly to the topic they need. The layout also separates the contents area from the reading area to reduce clutter.

Challenges and Lessons Learned

The AIS page was the main challenge because it needed to look interactive without becoming too complicated. I also had to make sure the background changes did not reduce readability. I solved that by keeping the content area in cards and using only three simple impact levels.

Accessibility Considerations

My pages use accessibility in slightly different ways. On the splash screen, I made sure the text stays readable on top of the background image by using contrast and spacing. On the AIS page, I added clear focus and selected states so users can understand the interaction. On the content page, I kept headings and section links straightforward so the page is easier to navigate.

Compliance

On my pages, responsible publishing meant keeping the interactive content honest and suitable for an educational site. I did not include exaggerated statistics or inappropriate media. I also used simple original wording for the action cards and made sure image use was relevant to the SDG topic rather than decorative only.
